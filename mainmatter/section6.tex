\section{Specific Architectures}

The following setup is common in current 21cm SBI:

One networks learns a compression of the image data to a much lower-dimensional latent space, i.e. a \red{summary statistic}. Another network learns a mapping from the latent space to the parameter space.

The compressing network usually has a much heavier architecture. Once it has been trained, one can learn different lighter networks to map its summary to different parameter spaces. Example for parameter spaces include the parameters used for the simulations, or a simple physical summary statistic of the image data.

One such construction is \textit{21cmPIE-INN}, which consists of a CNN acting as a compressing network and a normalizing flow mapping its output to six parameters used for the original simulations. The architecture is visualized in figure~\ref{fig:21cmPIE-INN_architecture}.

It is also possible to learn e.g. the reionization history with variable resolution from the CNN's summary, as is done with the \textit{EoRFlow} architecture~\red{Citation here}.
